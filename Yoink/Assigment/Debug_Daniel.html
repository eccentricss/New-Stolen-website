üìù Problem Description

A discount calculation function contains mathematical errors that produce incorrect results when calculating final prices after discounts.

üêõ Buggy Code


def calculate_discount(price, discount_percent):
    # ‚ùå BUG: Incorrect percentage calculation
    discount = price * discount_percent
    final_price = price - discount
    return final_price

# Test cases demonstrating the bug
print(calculate_discount(100, 20))   # Expected: 80.0, Actual: -1900.0
print(calculate_discount(50, 10))    # Expected: 45.0, Actual: -450.0

üîç Debugging Process

  1. Identify the Issue: The function treats discount_percent as a multiplier instead of percentage
  2. Root Cause: Missing division by 100 to convert percentage to decimal
  3. Impact: Results in dramatically incorrect prices

‚úÖ Fixed Solution


def calculate_discount(price, discount_percent):
    # ‚úÖ FIX: Convert percentage to decimal
    discount = price * (discount_percent / 100)
    final_price = price - discount
    return final_price

# Test cases - now working correctly
print(calculate_discount(100, 20))   # Result: 80.0 ‚úì
print(calculate_discount(50, 10))    # Result: 45.0 ‚úì
print(calculate_discount(200, 15))   # Result: 170.0 ‚úì

üí° Learning Outcomes

  • Always validate percentage calculations
  • Use test cases to verify expected behavior
  • Understand the difference between percentages and decimals
  • Systematic debugging approach
